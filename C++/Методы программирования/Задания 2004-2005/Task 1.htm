Задания для практических занятий по теме
«Структуры данных: стек и список»
Часть 1

1. Класс Stack имеет интерфейс
 
     Stack=class
              int Pop(TElement x); //извлечь x из стека
              int Push(TElement x); //поместить x в стек
              int IsEmpty(); //проверка стека на пустоту
              TElement Top(); //прочитать вершину
           end;
 
    1) Реализовать класс Stack на основе массива и на основе ссылочного списка (язык программирования  С++).
    2) Сравнить эффективность двух реализаций класса. Для этого сгенерировать последовательность операций со стеком из 107 действий (хранить в памяти). Затем проделать указанную последовательность операций с двумя реализациями стека, измеряя при этом продолжительность этой процедуры.
    3) Какие последовательности операций целесообразно использовать на шаге 2 для проверки производительности?
    4) Реализовать процедуру сортировки элементов стека, используя только интерфейс класса и переменные типа TElement (возможно использование нескольких стеков).
    5) Измерить скорость работы процедуры шага 4 для различных реализаций. Для этого подготовить тестовую программу, содержащую реализации стека и процедуры сортировки, а также производящую следующие действия:

  а) открытие файла data.txt и считывание в стек его содержимого (в файле хранятся 50000 целых чисел по одному в каждой строке);
  б) сортировка содержимого стека с одновременным замером затраченного на это время;
  в) повторить шаги а) и б) для второй реализации стека;
  г) вывод на экран сообщение вида

реализация с помощью массива: x секунд
реализация с помощью списка : y секунд

Точность измерений до 0.001 с.

    6*) Реализовать структуру данных массив стеков, содержащую k стеков с общим количеством элементов не более n. Во внутренней реализации разрешается использование не более трех целочисленных массивов и нескольких целочисленных переменных. Вызовы функций для операций со стеками должны быть организованы следующим образом:
     операция(<номер стека>[,<элемент>]);
Использование указателей не допускается.

Совпадение кусков исходного кода у нескольких студентов карается презрением на экзамене, а также плохой оценкой.
