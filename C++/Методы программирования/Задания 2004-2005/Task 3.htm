Префиксная и постфиксная запись арифметических выражений

Компилятор при анализе арифметических выражений преобразует их из префиксной записи в постфиксную.

Префиксной записью называется запись арифметической операции, в которой знак операции располагается между операндами. Это естественная запись, к которой мы все привыкли и умеем ее читать. Но математические выражения в префиксной записи не могут быть вычислены машиной "на лету", т.к. для этого необходим синтаксический разбор. Естественно, что каждый раз запуская программу это выражение не может анализироваться - слишком большой расход времени. Для вычисления математических выражений используется префиксная запись, т.е. запись при которой знак операции располагается после операндов.

Пример.
Префиксная запись: a + b
Постфиксная запсиь: a b +

По любому префиксному выражению можно построить его постфиксный эквивалент.

Более сложный пример.
Префиксная запись: (a+b)*c-d/(a+c)
Постфиксная запись: a b + c * d a c + / -

Правило преобразования.

Определим правило преобразования префиксной записи в постфиксную рекурсивно.

Пусть postfix(<выражение>) означает функцию преобразования префиксной записи <выражение> в постфиксный аналог, тогда имеет место соотношение

postfix(<операнд 1> <операция> <операнд 2>) =
    postfix(<операнд 1>) postfix(<операнд 2>) <операция>

Здесь <операция> - последняя арифметическая операция, выполняемая при вычислении значения выражения.

Таким образом, согласно введенному правилу
postfix((a+b)*c-d/(a+c)) =
    postfix((a+b)*c) postfix(d/(a+c)) ­ =
    postfix(a+b) c * d postfix(a+c) / - =
    a b + c * d a c + / -

Правило вычисления.

Допусти мы научились строить постфиксную запись. Теперь необходимо научиться вычислять арифметические выражение, записанные в постфиксной записи.

При вычислении используется стек, обозначим его S.

0) Установить текущую позицию просмотра постфиксной записи на первый элемент.
1) Просматриваем постфиксную запись слева направо и заносить в стек S все встречающиеся переменные до тех пор, пока нам не встретиться знак операции.
2) Извлечь из стека S два вершинных элемента и применить к ним операцию.
3) Затолкать в стек S полученное число.
4) Если постфиксная запись просмотрена до конца, то в S должно находиться единственное число - значение выражения. Останавливаемся.
5) Переходим к шагу 1).

Задание.

1. Обосновать правило преобразования.

2. Обосновать правило вычисления.

3. Реализовать процедуру преобразования префиксной записи числа в постфиксную.

  Будем предполагать, что изначально в префиксную запись входят операнды, обозначаемы маленькими латинскими буквами, знаки операций (+,-,*,/) и знаки расстановки приоритетов (круглые скобки). См. Более сложный пример. Дополнительно можно реализовать операцию ^ (возведение в степень) или другие бинарные операции.

Программа должна считывать исходное выражение из файла expression.txt и выводить его вместе с результатом преобразования на экран.

4. Дополним список арифметических операций ещё дополнительными функциями:
    а) "^" -- возведение в степень;
    б) тригонометрические функции (sin, cos, tan, cot, asin, acos, atan, acot);
    в) показательные функции (exp, ln);
    г) степенные функции (sqrt - квадратный корень, sqrt3 - кубический корень).
    г) что-нибудь на Ваш выбор.

5. Реализовать процедуру вычисления постфиксного выражения.

  Будем дополнительно предполагать, что в файле expression.txt помимо префиксной записи в строках, начиная со второй, расположены записи вида
<переменная>=<значение>
из которых необходимо получить значения переменных для подстановки в выражение.

При реализации использовать написанный ранее класс Stack.

Вычисленное значение и значения переменных необходимо вывести на экран.

6. Написать прикладную программу с возможностью ввода пользовательских функций, являющихся композицией основных.
Пример:
    построение графиков функций;
    исследование функций на минимум и максимум.
